# 生成AIメモリ技術

## 1. はじめに

LLMのステートレスな性質は、実運用において **「AIの健忘症（AI Amnesia）」** と揶揄される致命的な課題を引き起こす。
長期間にわたる対話や、数日をまたぐ複雑なタスクの実行において、以前提供した前提条件や指示をモデルが忘却してしまうことは、AIを単なる一問一答のツールから真の協働パートナー（Copilot）へと昇華させる上での最大のボトルネックとなる。

ここでは、この課題を解決するためのメモリシステム技術の基礎知識を、主にarXivにて公開されている論文を引用しながらまとめる。

## 2. メモリ技術概要

AIのメモリシステムは人間の記憶モデルのアナロジーを用いて分類されることが多い。
当該アナロジーはメモリシステムへの理解を深める入り口としてわかりやすいものであり、ここでも概要説明として触れる。
人間の記憶が感覚記憶、短期記憶、長期記憶に分かれるように、AIシステムにおいても情報を保持する期間とメカニズムに基づく厳密な分類が存在する。

### 2.1. 3D-8Qタクソノミー

LLMメモリの分類や人間との記憶モデルの比較に関する包括的な学術的サーベイとしては、『From Human Memory to AI Memory: A Survey on Memory Mechanisms in the Era of LLMs』(arXiv:2504.15965) が非常に詳細な見取り図を提供している[^1]。

| Object   | Form           | Time       | Quadrant | 役割                 | 説明                                                                                                                                                                                       |
| :------- | :------------- | :--------- | :------: | :------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Personal | Non-Parametric | Short-Term |    I     | ワーキングメモリ     | マルチターン対話のコンテキスト（セッション内の履歴）。ChatGPT の「今のチャット履歴」など。その場の文脈理解・応答の一貫性を支える。                                                         |
| Personal | Non-Parametric | Long-Term  |    II    | エピソード記憶       | セッションをまたぐユーザの長期記憶（プロフィール、好み、過去の会話要約など）。ChatGPT Memory, mem0 など。パーソナライズや長期的な一貫性を支える。                                          |
| Personal | Parametric     | Short-Term |   III    | ワーキングメモリ     | プロンプトキャッシュなど、特定ユーザのコンテキストをキャッシュして高速化する仕組み。そのユーザ向けの処理を一時的にパラメトリックに持っておくイメージ。                                     |
| Personal | Parametric     | Long-Term  |    IV    | 意味記憶             | 個人データでの微調整や Knowledge Editing によって、ユーザ固有情報をモデルパラメータに埋め込む。個人ごとの「パーソナル LLM」「キャラ LLM」など。                                            |
| System   | Non-Parametric | Short-Term |    V     | ワーキングメモリ     | ReAct, Reflexion などでタスク中に生成される思考ログ・計画・ツール呼び出し結果など。そのタスクをうまく解くための一時的な「思考のメモ」。                                                    |
| System   | Non-Parametric | Long-Term  |    VI    | 手続き記憶           | 過去タスクの成功例／失敗例・スキルライブラリ・ワークフロー・テンプレートなどを蓄積する。Voyager, Buffer of Thoughts など。「経験から学んだ戦略・スキル」を再利用する長期的なシステム記憶。 |
| System   | Parametric     | Short-Term |   VII    | ワーキングメモリ     | KV キャッシュ・量子化・圧縮など、推論時の一時パラメトリックメモリ管理。vLLM, ChunkKV など。推論コストやレイテンシ削減といったシステム性能面のメモリ。                                      |
| System   | Parametric     | Long-Term  |   VIII   | 意味記憶＋手続き記憶 | 事前学習済み LLM 自体のパラメータに埋め込まれた知識・スキル、および Knowledge Editing/MemoryLLM/WISE などによる「継続的な知識更新」。世界知識・タスク知識を蓄積した長期記憶。              |

上記見取り図は論文のコアである、AIメモリを次の3軸で整理した**3D-8Q メモリ分類** である。

1. **Object（対象）**
  - Personal Memory：ユーザ固有の情報・履歴・好みなど
  - System Memory：タスク途中の中間結果やスキル、ワークフローなど、システム内部の記憶
2. **Form（形態）**
   - Non-Parametric：外部に保存されたメモリ（DB, ベクターストア, グラフ, ドキュメントなど）
   - Parametric：モデルパラメータとして埋め込まれたメモリ（事前学習知識や微調整で埋め込んだ個人情報など）
3. **Time（時間スケール）**
   - Short-Term：セッション内／その場限りの短期的な文脈（コンテキスト）
   - Long-Term：セッションをまたいで保持される長期メモリ

### 2.2. 短期記憶と長期記憶について

3D-8Qタクソノミーにおける、主に **Time（時間スケール）** についてピックアップする。

#### 短期記憶

**「短期記憶（Short-term Memory）」** または　**「ワーキングメモリ」** は、一般的にコンテキストウィンドウ内に保持される情報と定義される。

これは現在のセッションやタスクの実行中にのみ有効な活性化された情報であり、In-context Learning（文脈内学習）の基盤となる。
モデルはファインチューニングを伴わずとも、この短期記憶内に提示されたプロンプトの指示や少数の例示（Few-shot）から動的にパターンを学習し、出力を適応させることができる。

#### 長期記憶

**「長期記憶（Long-term Memory）」** は、一般的にセッションをまたいで永続化される情報と定義される。

これにはさらに二つの形態が存在する。

1. **パラメトリックメモリ（Parametric Memory）**: モデルの重みパラメータ自体に焼き込まれた知識
2. **非パラメトリックメモリ（External Memory）**: ベクトルデータベースやナレッジグラフなどの外部ストレージに保存される知識

現代の高度なAIエージェント設計においては、膨大な情報を外部ストレージに長期記憶として保持し、現在のタスクに関連する情報のみを動的に検索（Retrieve）して短期記憶（コンテキストウィンドウ）に注入するハイブリッドアーキテクチャが主流となっている。
また、対象次元の観点から、システム全体の知識を司る **「システムメモリ」**と、特定のユーザーの嗜好や履歴を追跡する「**パーソナルメモリ（エピソード記憶）」**という分類も、パーソナライズされたAIの文脈では重要視されている。

### 2.3. 評価指標

メモリシステムを設計・評価する際、システムは以下の主要な評価指標の複雑なトレードオフを管理しなければならない。

| 指標                       | 焦点                   |
| :------------------------- | :--------------------- |
| **精度 (Accuracy)**        | 検索・回答の正確性     |
| **レイテンシ (Latency)**   | 応答速度               |
| **コスト (Cost)**          | トークン消費・計算資源 |
| **情報の鮮度 (Freshness)** | 古い情報の更新能力     |

ここに示した4指標はあくまでごく一部最低限のものであり、これらだけでは不十分である。
ここでは脚注に記載の論文紹介にとどめるが、デプロイ後のインタラクションから新しいスキルや振る舞いを学ぶ能力、ミリオン級のコンテキスト全体理解に基づく質問に答える能力、古い知識を適切に上書き・忘却する能力、など、この後も紹介するAI エージェントに必要な能力を測る様々な指標が必要とされている[^2] [^3] [^4] [^5]。

### 2.4. 近年の生成AIメモリに至るまでの技術変遷

#### Transformerアーキテクチャはステートレス

近年の大規模言語モデル（LLM）の基盤をなす **Transformerアーキテクチャ** は、 **本質的にステートレス（状態を持たない）** なシステムとして設計されている。
入力されたプロンプトに対して確率的に次のトークンを予測する関数に過ぎず、モデル自体が過去の対話履歴やユーザー固有の文脈を自律的に保持することはない。
この構造的制約を克服し、人工知能に連続性と文脈理解をもたらすための技術群が、生成AIにおける「メモリ技術」である。

ではなぜTransformerアーキテクチャはステートレストいう弱点を持っているのか？
理由は、Transformerはメモリを犠牲にしてスケーラビリティを手に入れた歴史があるためだ。

#### RNNアーキテクチャ時代

Transformerアーキテクチャは、**RNN(Recurrent Neural Networks)**の弱点を補う中で生まれたアーキテクチャである。

RNNは時系列データや自然言語のように「順序」に意味があるデータを扱うための画期的な構造であった。
**ステートフルなアーキテクチャ**であり、過去の情報を「隠れ状態（Hidden State）」という一つのベクトルに圧縮して次へ次へと受け渡していくことで、擬似的な「記憶」を形成していた。

$$h_t = f(W_h \cdot h_{t-1} + W_x \cdot x_t)$$

>　前の時刻の状態 $h_{t-1}$ を次の計算に渡す → これが「記憶」

ただし、以下の課題を抱えていた。

- 勾配消失問題により、長い文章の最初の方の情報を忘れてしまう
- 逐次処理のため**並列計算ができず、学習が遅い**

これらの課題を補うために、LSTM, Seq2Seqなどの手法が登場、CNN(畳み込みニュートラルネットワーク)の研究が進んだ。

その中で、**Attentionが誕生**した。
これはざっくり言うと、**「出力する単語に関連する入力単語をその都度見に行く（注意を向ける）」もの**であり、長い文章を1つの固定長ベクトルに詰め込む必要を排除する工夫だった。
当初これは、RNNの補助輪的に扱われていた。

#### Attention Is All You Need

2017年、Googleの論文「Attention Is All You Need」が登場した。
このコンセプトを端的に言うと、「RNNもCNNもいらない、Attentionだけで十分」というものである。

具体的には、以下のような特徴を持つ。

- **逐次処理を完全に排除し、文章を一気に並列処理できるように**した。
- Self-Attentionとして、**文章内の全単語間の関係を直接計算**。どれだけ離れた単語でも一瞬で関連性を抽出できる反面、過去の情報を覚えているのではなく処理のために過去の全データを読み直す仕組み。

つまり、処理速度の代償として記憶を失う、ただし、「**記憶（状態の保持）を頑張るのではなく、全データに直接アクセス（Attention）できるようにすれば、記憶の劣化問題は解決する**」ということだ。

#### 当時からわかっていたTransformerの課題

しかし、そう全てがうまくはいかない。
計算量がシーケンス長の2乗（ $n^2$ ）で増えるため、一度に扱える文章の長さには上限がある。
そのため、「数冊前の本の内容を覚えている」といった**超長期的な記憶は、当時のオリジナルのTransformer単体では構造上想定することができなかった**。

こうして、メモリ技術が必要になっていく。
