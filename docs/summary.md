# 生成AIメモリ技術

## メモリ技術概要

学術的および認知心理学的な観点から、AIのメモリシステムは人間の記憶モデルのアナロジーを用いて分類されることが多い。
人間の記憶が感覚記憶、短期記憶、長期記憶に分かれるように、AIシステムにおいても情報を保持する期間とメカニズムに基づく厳密な分類が存在する。

### 3D-8Qタクソノミー

LLMメモリの分類や人間との記憶モデルの比較に関する包括的な学術的サーベイとしては、『From Human Memory to AI Memory: A Survey on Memory Mechanisms in the Era of LLMs』(arXiv:2504.15965) が非常に詳細な見取り図を提供している[^1]。

| Object   | Form           | Time       | Quadrant | 役割                 | 説明                                                                                                                                                                                       |
| :------- | :------------- | :--------- | :------: | :------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Personal | Non-Parametric | Short-Term |    I     | ワーキングメモリ     | マルチターン対話のコンテキスト（セッション内の履歴）。ChatGPT の「今のチャット履歴」など。その場の文脈理解・応答の一貫性を支える。                                                         |
| Personal | Non-Parametric | Long-Term  |    II    | エピソード記憶       | セッションをまたぐユーザの長期記憶（プロフィール、好み、過去の会話要約など）。ChatGPT Memory, mem0 など。パーソナライズや長期的な一貫性を支える。                                          |
| Personal | Parametric     | Short-Term |   III    | ワーキングメモリ     | プロンプトキャッシュなど、特定ユーザのコンテキストをキャッシュして高速化する仕組み。そのユーザ向けの処理を一時的にパラメトリックに持っておくイメージ。                                     |
| Personal | Parametric     | Long-Term  |    IV    | 意味記憶             | 個人データでの微調整や Knowledge Editing によって、ユーザ固有情報をモデルパラメータに埋め込む。個人ごとの「パーソナル LLM」「キャラ LLM」など。                                            |
| System   | Non-Parametric | Short-Term |    V     | ワーキングメモリ     | ReAct, Reflexion などでタスク中に生成される思考ログ・計画・ツール呼び出し結果など。そのタスクをうまく解くための一時的な「思考のメモ」。                                                    |
| System   | Non-Parametric | Long-Term  |    VI    | 手続き記憶           | 過去タスクの成功例／失敗例・スキルライブラリ・ワークフロー・テンプレートなどを蓄積する。Voyager, Buffer of Thoughts など。「経験から学んだ戦略・スキル」を再利用する長期的なシステム記憶。 |
| System   | Parametric     | Short-Term |   VII    | ワーキングメモリ     | KV キャッシュ・量子化・圧縮など、推論時の一時パラメトリックメモリ管理。vLLM, ChunkKV など。推論コストやレイテンシ削減といったシステム性能面のメモリ。                                      |
| System   | Parametric     | Long-Term  |   VIII   | 意味記憶＋手続き記憶 | 事前学習済み LLM 自体のパラメータに埋め込まれた知識・スキル、および Knowledge Editing/MemoryLLM/WISE などによる「継続的な知識更新」。世界知識・タスク知識を蓄積した長期記憶。              |

上記見取り図は論文のコアである、AIメモリを次の3軸で整理した**3D-8Q メモリ分類** です。

1. **Object（対象）**
  - Personal Memory：ユーザ固有の情報・履歴・好みなど
  - System Memory：タスク途中の中間結果やスキル、ワークフローなど、システム内部の記憶
2. **Form（形態）**
   - Non-Parametric：外部に保存されたメモリ（DB, ベクターストア, グラフ, ドキュメントなど）
   - Parametric：モデルパラメータとして埋め込まれたメモリ（事前学習知識や微調整で埋め込んだ個人情報など）
3. **Time（時間スケール）**
   - Short-Term：セッション内／その場限りの短期的な文脈（コンテキスト）
   - Long-Term：セッションをまたいで保持される長期メモリ

### 短期記憶と長期記憶

#### 短期記憶

**「短期記憶（Short-term Memory）」**または　**「ワーキングメモリ」** を、コンテキストウィンドウ内に保持される情報と定義する。

これは現在のセッションやタスクの実行中にのみ有効な活性化された情報であり、In-context Learning（文脈内学習）の基盤となる。
モデルはファインチューニングを伴わずとも、この短期記憶内に提示されたプロンプトの指示や少数の例示（Few-shot）から動的にパターンを学習し、出力を適応させることができる。

#### 長期記憶

**「長期記憶（Long-term Memory）」**を、セッションをまたいで永続化される情報と定義する。

これにはさらに二つの形態が存在する。

1. **パラメトリックメモリ（Parametric Memory）**: モデルの重みパラメータ自体に焼き込まれた知識
2. **非パラメトリックメモリ（External Memory）**: ベクトルデータベースやナレッジグラフなどの外部ストレージに保存される知識

現代の高度なAIエージェント設計においては、膨大な情報を外部ストレージに長期記憶として保持し、現在のタスクに関連する情報のみを動的に検索（Retrieve）して短期記憶（コンテキストウィンドウ）に注入するハイブリッドアーキテクチャが主流となっている。
また、対象次元の観点から、システム全体の知識を司る **「システムメモリ」**と、特定のユーザーの嗜好や履歴を追跡する「**パーソナルメモリ（エピソード記憶）」**という分類も、パーソナライズされたAIの文脈では重要視されている。
